Blocks

You and your little brother love playing with numbered blocks. You currently have N blocks lined up and sorted in front of you, labeled 1, 2, 3, ..., N, with. You and your brother came up with a game during which you can do one of two things to the blocks:

  1. Move the leftmost block in the starting line and add it to the top of the tower
  2. Take the top block off of the tower and place it at the end of a different line of blocks

If you have no blocks left, you have to take blocks off the tower. If you have no blocks on your tower, then you have to put one on. The game ends once every block is in the new line.

Your brother decided to challenge you. He arranged the blocks 1 to N into some configuration. According to him, you'll win if you can create that configuration by playing the game. If you can't, you lose.

Input

The first line of input will contain a single integer T, the number of test cases. The first line of each test case has an integer, N. The next line has N integers, the ordering of the blocks after your brother rearranged them.

Constraints

There are at most 10 test cases. There will be no more than 1,000,000 blocks per test case.

Output

For each test case, if the end ordering is not possible, print NO on a single line. If it is possible, print YES on a single line, and then print the operations STACK and PLACE required to make that end ordering.

Sample Input

2
3
2 1 3
3
3 2 1

Sample Output

YES
STACK

PLACE
